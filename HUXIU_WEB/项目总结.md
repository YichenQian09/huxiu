# 项目总结
## 技术上的收获
### js
1. 熟悉了一下jquery选择器的使用，parent是选择元素的父元素，parents是选择元素的祖先元素；children是选择元素的子元素，find可以选择元素的后代。index可以获取元素是同类元素中第几个。$(this)和这些选择器时常要配合在一起使用。
2. 使用ajax从后台获取数据，并动态添加元素到网页上，可以与后台约定接口，并以
json数据的形式传送数据,根据后台传来的json判断如果没有更多数据，就将加载按钮隐藏显示到底了。
3. 练习了对插件的使用和按照需求修改
4. 在对一些js的小的效果（如点赞，二维码分享）的实现中，js书写的思路逻辑更加清晰，练习了使用jquery

### css
1. css阴影效果的实现，四个参数分别指的是阴影水平偏移量，阴影竖直偏移量，模糊半径，扩展半径，可以调整各个参数来获得最好的效果。
```
      -webkit-box-shadow:0 0 30px 3px rgb(241, 241, 241);  
  	-moz-box-shadow:0 0 30px 3px rgb(241, 241, 241);  
  	box-shadow:0 0 30px 3px rgb(241, 241, 241); 
```
2. 考虑到后台的数据量和图片大小不定，设置css样式。

## 心得体会
1. 在UI设计稿出来后，可以对比需求分析，好好讨论一下各个地方需求的实现要求，以避免在项目进行中发现UI 前端 程序理解实现不一样的情况。
2. 在项目过程中，彼此间的合作需要互相帮助，互相包容。
3. 之前还天真的以为前端一个多星期写完后就可以甩锅……与UI，与程序交流后又会发现有种种问题，不停地修改，所以一直到项目截止都有些凌乱，需要一颗强大的心脏，良好的沟通能力，随机应变的能力，又出现问题时要镇静地分析思考合作解决……
4. 觉得自己还是不怎么了解程序的工作，但我相信我的队友对前端有了更深一步的了解。
5. 哪怕前端写的不完善，也应该尽早给UI看，尽早解决一些展示效果上的问题，而不是一直拖到最后程序绑完代码时。
